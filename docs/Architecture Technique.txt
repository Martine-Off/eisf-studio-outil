## ⚙️ Backend

| Élément | Valeur |
|---|---|
| **Fichier d'entrée** | `server/server.js` |
| **Framework** | Express.js (v4.21) |
| **Langage** | JavaScript (CommonJS / `require`) |
| **Port** | `3001` (configurable via `PORT` dans `.env`) |
| **Commande dev** | `npm run dev` (nodemon avec auto-reload) |
| **Commande prod** | `npm start` → `node server.js` |
| **Sert le frontend ?** | ❌ **Non** — API uniquement (`/api/*`) |

### Routes API

Toutes les routes sont préfixées par `/api` :

| Route | Fichier | Description |
|---|---|---|
| `/api/auth` | `routes/auth.js` | Inscription, connexion (JWT) |
| `/api/projects` | `routes/projects.js` | CRUD projets, upload .docx |
| `/api/podcasts` | `routes/podcasts.js` | Gestion des podcasts |
| `/api/dialogues` | `routes/dialogues.js` | Édition des lignes de dialogue |
| `/api/ai` | `routes/ai.js` | Génération IA (script, vérification, audio) |
| `/api/export` | `routes/export.js` | Export PDF et fichiers |
| `/health` | `server.js` | Health check (hors préfixe `/api`) |

### Authentification

- **JWT** (JSON Web Token) via `jsonwebtoken`
- Token stocké côté client dans `localStorage`
- Middleware `middleware/auth.js` vérifie le Bearer token sur chaque requête protégée
- Secret défini par `JWT_SECRET` dans `.env`

## 🔗 Connexions Frontend ↔ Backend

### En développement

Le frontend utilise un **proxy Vite** pour rediriger les appels API :

```typescript
// vite.config.ts
server: {
  port: 5173,
  proxy: {
    '/api': {
      target: 'http://localhost:3001',
      changeOrigin: true,
    },
  },
}
```

Le client Axios appelle simplement `/api/*` (URL relative) :

```typescript
// src/utils/api.ts
const api = axios.create({
    baseURL: '/api',   // Proxy Vite redirige vers localhost:3001
    timeout: 300000,   // 5 minutes (pour les appels IA longs)
});
```

### En production

> [!IMPORTANT]
> En production, il n'y a **pas de proxy Vite**. Il faut soit :
> 1. **Configurer un reverse proxy** (Nginx, Apache, Caddy) pour router `/api/*` vers le backend
> 2. **Ou modifier `baseURL`** dans `api.ts` pour pointer vers l'URL absolue du backend
> 3. **Et configurer `ALLOWED_ORIGINS`** dans le `.env` serveur avec le domaine du frontend

## 🔐 Variables d'environnement

### Backend (`server/.env`)

| Variable | Description | Exemple |
|---|---|---|
| `PORT` | Port du serveur Express | `3001` |
| `DATABASE_URL` | Connection string PostgreSQL | `postgresql://postgres:PASSWORD@localhost:5433/studio_eisf` |
| `JWT_SECRET` | Secret pour signer les JWT | *(chaîne secrète)* |
| `OPENAI_API_KEY` | Clé API OpenAI | `sk-proj-...` |
| `GEMINI_API_KEY` | Clé API Google Gemini | `AIza...` |
| `N8N_WEBHOOK_URL` | URL webhook n8n (optionnel) | `http://localhost:5678/webhook/...` |
| `USE_MOCK_DB` | Utiliser la DB en mémoire | `false` |
| `USE_MOCK_AI` | Simuler les appels IA | `false` |
| `ALLOWED_ORIGINS` | Origines CORS autorisées (prod) | `https://studio.eisf.fr` |
| `NODE_ENV` | Environnement | `production` ou absent |

### Frontend (`client/.env`)

| Variable | Description | Exemple |
|---|---|---|
| `VITE_API_URL` | URL de base du backend | `http://localhost:3001` |

## 🚀 Lancement complet (développement)

### Prérequis

- **Node.js** ≥ 18
- **Docker** (pour PostgreSQL)
- Fichiers `.env` configurés dans `client/` et `server/`

### Étape par étape

```bash
# 1️⃣ Lancer PostgreSQL via Docker
docker-compose up -d

# 2️⃣ Installer les dépendances
cd server && npm install
cd ../client && npm install

# 3️⃣ Initialiser la base de données (1ère fois uniquement)
cd ../server
node initDB.js          # Crée les tables
node initUser.js        # Crée l'admin (admin@eisf.fr / admin)
node migrate.js         # Applique les migrations

# 4️⃣ Lancer le backend (terminal 1)
cd server
npm run dev             # → http://localhost:3001

# 5️⃣ Lancer le frontend (terminal 2)
cd client
npm run dev             # → http://localhost:5173
```

### Vérification

- **Health check** : `GET http://localhost:3001/health`
- **Frontend** : Ouvrir `http://localhost:5173` dans le navigateur
- **Connexion** : `admin@eisf.fr` / `admin`

## 🚀 Déploiement en production

### Option recommandée : VPS avec Nginx

```bash
# 1️⃣ Build du frontend
cd client
npm run build           # Génère client/dist/

# 2️⃣ Lancer le backend
cd server
NODE_ENV=production npm start    # → port 3001
```

### Configuration Nginx (exemple)

```nginx
server {
    listen 80;
    server_name studio.eisf.fr;

    # Frontend (fichiers statiques)
    location / {
        root /chemin/vers/client/dist;
        try_files $uri $uri/ /index.html;
    }

    # API → Proxy vers Express
    location /api/ {
        proxy_pass http://localhost:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_http_version 1.1;
        proxy_read_timeout 300s;    # Important pour les appels IA longs
    }

    # Uploads statiques
    location /uploads/ {
        proxy_pass http://localhost:3001;
    }

    # Health check
    location /health {
        proxy_pass http://localhost:3001;
    }
}
```

> [!IMPORTANT]
> **En production, pensez à :**
> - Mettre un vrai `JWT_SECRET` (pas "martine" !)
> - Configurer `ALLOWED_ORIGINS` avec votre domaine
> - Utiliser un process manager comme **PM2** pour le backend : `pm2 start server.js --name studio-eisf`
> - Configurer SSL avec Let's Encrypt
> - Sécuriser/renouveler les clés API (OpenAI, Gemini)
