## 🎯 VISION PRODUIT

### Objectif
Transformer automatiquement des cours Storyline (format .docx) en mini-podcasts pédagogiques (4-7 min) avec dialogue naturel entre deux personnages : Anabelle et Bryan.

### Problème résolu
- Réduire le temps de production de contenus audio pédagogiques (de 2h/cours à 15 min)
- Offrir un format d'apprentissage accessible (audio pour trajets, révisions)
- Maintenir la qualité pédagogique des cours EISF

### Public cible
- **Utilisateurs principaux** : Professeurs et ingénieurs pédagogiques EISF
- **Bénéficiaires finaux** : Étudiants EISF (apprentis boulangers, pâtissiers)

---

## 👥 PERSONNAGES DU PODCAST

### Anabelle (70% du temps de parole)
- **Rôle** : Experte EISF, enseignante
- **Ton** : Bienveillante, pédagogue, structurée
- **Style** : Tutoiement, utilise "on", explications claires
- **Fonction** : Transmet le contenu principal du cours

### Bryan (30% du temps de parole)
- **Rôle** : Apprenant type "NotebookLM"
- **Ton** : Curieux, spontané, ancre dans le quotidien
- **Style** : Interpelle ("Attends..."), reformule, fait des liens concrets
- **Fonction** : Facilite la compréhension, pose les questions que l'étudiant se pose

### Ratio de parole
- **Anabelle** : 70% (contenu pédagogique principal)
- **Bryan** : 30% (reformulations, questions, validation)

## 📊 FORMAT PODCAST

### Contraintes de durée
- **Durée cible** : 4-7 minutes par podcast
- **Durée minimum** : 4 minutes
- **Durée maximum** : 7 minutes
- **Calcul** : 150 mots/minute (norme TTS)

### Structure type (4-7 min)

```
1. JINGLE (15 secondes)
   "Ceci est un podcast produit à partir des cours originaux de l'EISF."
   Anabelle : "Salut Bryan !"
   Bryan : "Salut Annabelle ! C'est parti ?"
   Anabelle : "C'est parti. Aujourd'hui : [TITRE CHAPITRE]."

2. INTRO RAPIDE (20-30 secondes)
   → Anabelle annonce le sujet
   → Bryan fait un lien avec le quotidien

3. CONTENU PRINCIPAL AVEC QUIZ INTÉGRÉ (3-5 minutes)
   → Anabelle explique les concepts (70%)
   → Bryan questionne, reformule, valide (30%)
   → Quiz intégré naturellement dans le dialogue
   
4. CONCLUSION (20-30 secondes)
   → Anabelle récapitule en 1 phrase
   → Bryan valide la compréhension
   → Transition ("À tout à l'heure !")

Total : 4-7 minutes
```

### Principe de découpage
- **1 podcast = 1 chapitre ou sous-chapitre Storyline**
- **Découpage automatique** : Mode chronologique (suit l'ordre du cours)
- **Validation manuelle** : L'utilisateur peut ajuster le découpage proposé

## 🛠️ FONCTIONNALITÉS DÉTAILLÉES

### ÉTAPE A : IMPORT & PARSING

#### Import fichier
- **Format accepté** : .docx (export Storyline)
- **Taille max** : 50 MB
- **Méthode** : Drag & Drop ou sélection fichier
- **Bibliothèque** : mammoth.js (lecture Word)

#### Parsing automatique
```javascript
Structure détectée :
- ID Storyline (ignoré)
- Type de zone (filtré : garde uniquement contenu pédagogique)
- Texte original (colonne utilisée)
- Traduction (colonne utilisée)
```

#### Filtrage intelligent
**Conservé** :
- Zones de texte avec contenu pédagogique (>20 caractères)
- Paragraphes explicatifs
- Définitions et concepts

**Ignoré** :
- Métadonnées Storyline (IDs, dates, auteur)
- Éléments de navigation ("Commencer", "Suivant")
- Titres de diapositives vides
- États graphiques (Normal/Survol) sans contenu

#### Détection chapitres
L'IA détecte automatiquement les sections principales :
- Titres courts (<50 caractères)
- Changements de thématique
- Numérotation (1-, 2-, 3-...)

#### B2 - Génération dialogue Anabelle/Bryan

**Prompt Gemini** (voir PROMPTS_GEMINI.md) génère :

```
Format de sortie :

[JINGLE]
Anabelle : "Ceci est un podcast produit à partir des cours originaux de l'EISF. Salut Bryan !"
Bryan : "Salut Annabelle ! C'est parti ?"
Anabelle : "C'est parti. Aujourd'hui : La farine."

[INTRO]
Anabelle : "Alors Bryan, aujourd'hui on va parler des farines. Tu sais, c'est la base de tout en boulangerie !"
Bryan : "Carrément ! D'ailleurs moi au supermarché, y'a tellement de types de farine, je sais jamais laquelle prendre..."

[CONTENU]
Anabelle : "Excellente question ! Déjà, la farine, c'est quoi ? C'est un produit amylacé, c'est-à-dire une poudre qui vient de végétaux."
Bryan : "Donc en gros, on écrase des graines et ça fait de la farine ?"
Anabelle : "Exactement ! En pâtisserie, quand on dit 'farine' sans préciser, on parle de farine de blé."
[...]

[CONCLUSION]
Anabelle : "Voilà Bryan, retiens bien : farine = blé tendre par défaut. Pour les autres, on précise toujours."
Bryan : "OK pigé ! Merci Annabelle, à tout à l'heure !"
```

**Contraintes de génération** :
- Ratio 70/30 (Anabelle/Bryan)
- Jingle obligatoire (texte exact fourni)
- Ton naturel, conversationnel
- Quiz intégré dans le dialogue (pas de section séparée)
- Durée cible : 600-1050 mots (4-7 min)

#### B3 - Normalisation automatique

L'IA applique automatiquement ces transformations :

**Chiffres → Lettres**
```
150 → cent cinquante
2024 → deux mille vingt-quatre
3,5 kg → trois virgule cinq kilogrammes
```

**Acronymes → Forme phonétique**
```
EISF → É-I-S-F
DLC → Dé-El-Cé
BTS → Bé-Té-S
```

**Symboles → Texte**
```
& → et
% → pourcent
€ → euros
```

#### B4 - Double version générée

**Version 1 : Studio (pour enregistrement voix)**
```
Anabelle : "L'EISF (É-I-S-F) forme chaque année 150 (cent cinquante) apprentis."
```

**Version 2 : Lecture (pour entendants)**
```
Anabelle : "L'EISF forme chaque année 150 apprentis."
```

### ÉTAPE C : VÉRIFICATION IA (Auto-correction)

L'IA analyse automatiquement le script généré et corrige :

#### Vérifications effectuées

**1. Fidélité au contenu source**
```
Comparer dialogue généré VS fichier .docx original
→ Détecte les omissions (concepts non mentionnés)
→ Détecte les ajouts (inventions de l'IA)
→ Détecte les erreurs factuelles
```

**2. Corrections orthographiques**
```
Grammaire française
Accords
Conjugaisons
```

**3. Normalisation phonétique**
```
Vérifie que TOUS les chiffres sont en lettres
Vérifie que TOUS les acronymes ont leur forme phonétique
```

#### Rapport de fidélité

```
┌─────────────────────────────────────────┐
│ 📊 Rapport de fidélité au cours        │
│                                         │
│ 🟢 Fidélité : 92%                      │
│                                         │

│ Détails :                               │
```

**Code couleur** :

## 🗂️ GESTION FICHIERS & NETTOYAGE

### Cycle de vie des fichiers

```
1. ACTIF (en cours de travail)
   → Aucune limite de durée
   → Fichier .docx source conservé

2. INACTIF 30 JOURS (pas ouvert depuis 30j)
   → Popup : "Archiver le fichier source (.docx) ?"
   ┌─────────────────────────────────────────┐
   │ ⚠️ Ce projet est inactif depuis 30 jours│
   │                                         │
   │ Le fichier source (.docx) occupe 2.3 MB│
   │ Voulez-vous l'archiver ?                │
   │                                         │
   │ [Conserver] [Archiver] [Supprimer]     │
   └─────────────────────────────────────────┘
   
   → Si "Archiver" : Compression + stockage froid

3. ARCHIVÉ 90 JOURS
   → Avertissement : "Suppression définitive le [date]"
   → Email rappel J-7

4. SUPPRESSION AUTOMATIQUE
   → Après 120 jours total (30 + 90)
   → Conserve les exports (.docx, JSON)
   → Supprime uniquement le fichier source
```

**Pourquoi ce système ?**
- Libérer espace serveur/cache
- Protéger les projets actifs
- Donner le contrôle à l'utilisateur

## 📊 SCHÉMA BASE DE DONNÉES

### Tables principales

**users**
```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  first_name VARCHAR(100),
  last_name VARCHAR(100),
  created_at TIMESTAMP DEFAULT NOW(),
  last_login TIMESTAMP
);
```

**projects**
```sql
CREATE TABLE projects (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id),
  title VARCHAR(255) NOT NULL,
  status VARCHAR(20) DEFAULT 'draft', -- draft, in_progress, finalized
  source_file_path VARCHAR(500),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  last_opened_at TIMESTAMP
);
```

**podcasts**
```sql
CREATE TABLE podcasts (
  id SERIAL PRIMARY KEY,
  project_id INTEGER REFERENCES projects(id),
  title VARCHAR(255) NOT NULL,
  order_index INTEGER,
  word_count INTEGER,
  duration_seconds INTEGER,
  fidelity_score DECIMAL(5,2), -- 0.00 à 100.00
  created_at TIMESTAMP DEFAULT NOW()
);
```

**dialogues**
```sql
CREATE TABLE dialogues (
  id SERIAL PRIMARY KEY,
  podcast_id INTEGER REFERENCES podcasts(id),
  order_index INTEGER,
  character VARCHAR(50), -- 'anabelle' ou 'bryan'
  text_studio TEXT NOT NULL,
  text_reading TEXT NOT NULL,
  duration_seconds INTEGER,
  section VARCHAR(50) -- 'jingle', 'intro', 'content', 'conclusion'
);
```

**project_shares**
```sql
CREATE TABLE project_shares (
  id SERIAL PRIMARY KEY,
  project_id INTEGER REFERENCES projects(id),
  shared_with_user_id INTEGER REFERENCES users(id),
  permission VARCHAR(20), -- 'read_only', 'edit'
  created_at TIMESTAMP DEFAULT NOW()
);
```
